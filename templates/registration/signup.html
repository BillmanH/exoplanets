An important note about passwords:

I'm using Django's native password management, not my own system (which I think is the more secure choice). That being said, I am the sole owner and super user of the infrastructure so I could, in theory, unhash your password. I've never done this and have no curiosity about it. Also, I'll never ask for money on this site or for any personally identifying information. I just need a user/password system to save content and preferences.

Additionally I'll encourage you to read more about Django's password management process

Passwords are hashed and stored in a separate PostgreSQL database completely outside of the game’s ecosystem. The only account data that is stored in the game’s database graph is the ‘username’ field, which is used to map the logged in user to the game. There are no individual user metrics, or attempts to identify the logged in user in any way, nor will there ever be. That’s just not within my curiosity for this game.
